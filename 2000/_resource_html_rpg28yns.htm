·In-game messages can use placeholder values ("%X"). Placeholder values are replaced by the actual values by the engine.

·The meaning of placeholder values differs between each message. A short explanation is provided above each field.

·The following placeholder values exist:

o%S: Subject

o%O: Object

o%V: Value

o%U: Unit

·Example:

oFor the "Critical Hit on Actor/Enemy" messages, the placeholders %S and %O are defined as the attacker's name and the target's name, respectively.

oIf you set the message in the editor to "Ouch, %S hit %O on the head!", and a "Slime" monster is attacking your "Alex" actor, the actual message displayed in-game would be "Ouch, Slime hit Alex on the head!".

·It doesn't matter whether a battle message fits into a line after the placeholders have been replaced, because the system will automatically word-wrap them.

 

 

When a project is opened which has been created using a previous version of RPG Maker 2000 which didn't support placeholders, you will be prompted to upgrade the engine. In this case, all messages will automatically be modified to use the new placeholders.

 

[Glossary 1] [Glossary 2] tabs

·Set the messages that will be displayed during battle scenes. Enter the text to be shown as applicable to the listed situations.

·The field is greyed out where text (character names, etc.) and values (damage, etc.) will be automatically inserted into the message.

·The options are categorized by purpose as follows:

[Battle messages 1]: Messages displayed when battles start and end.

[Battle messages 2]: Messages to announce characters' actions.

[Battle messages 3]: Messages to announce the effects of attacks, as well as the items used

[Battle messages 4]: Messages displayed to alert the player to a change in character stats.

 

[Glossary 3] [Glossary 4] tabs

·Set the messages that will be displayed on the screen during shop and inn events.

·Messages set under [Shop messages: Type A] to [Type C] are used in [Shop Processing] events. Enter the terms and messages as appropriate in the fields.

·Messages set under [Inn messages: Type A] and [Type B] are used in [Inn Processing] events. Enter the terms and messages as appropriate in the fields.

·[Shop Parameters] are used in [Shop Processing] events to display the number of items possessed and equipped as well as monetary values. [Currency] is also used when [Inn Processing] events are carried out (among other situations where the currency unit will be displayed, including the %U placeholder in the "Money obtained" message).

 

[Glossary 5] tab

·Set the glossary for game system text such as stat names, menu and battle commands, save/load terms, etc.

·Under [Stats and Equipment Vocab], set the names for character stats and equipment slots.

o(Abbr.) denotes the abbreviated name for the term on the menu and other places.

oThese terms are also used as values for the %U placeholders in some of the messages.

·Under [Event Commands], set the names for menu commands (e.g. battle, event, and main menus).

·Under [Save/Load/To title], set the names and messages for the commands displayed on the save/load screens.
